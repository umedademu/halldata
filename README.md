パチスロ店のデータを収集するアプリ

現在の叩き台版では、`apps/gui` の画面から MJアリーナ箱崎店 の対象日データを取得できます。
先に登録店舗から対象店舗を選び、対象期間を入力し、期間の終了日を基準に機種一覧を読み込み、チェックした機種をまとめて取得する流れです。
機種一覧の読込直後は全機種が選択済みになり、取得完了時は初期状態で音を鳴らします。
また、`登録店舗` タブでURLから店舗名を自動取得して登録でき、登録店舗一覧は `Supabase` に保存して次回起動時も読み込みます。
店舗URLは保存前に表記ゆれをそろえて扱うため、同じ店舗URLで重複した店舗行が増えにくいようにしています。
取得結果は、1列目を日付、その右に台番ごとの比較列を並べる形で表示します。
取得中は、日付ごとのデータ取得が終わるたびに `local_data` へのローカル保存と `Supabase` への保存を自動で行います。
取得に失敗した場合は、画面で指定した秒数だけ休止してから最大3回まで再試行します。初期値は10秒です。
再取得時は、すでに保存済みの同一店舗、同一日付、同一機種の組み合わせを `local_data` と `Supabase` から確認し、取得をスキップします。
また、`データ取得` タブから `サイトセブン` の固定対象として `Ａパーク春日店` の `ネオアイムジャグラーEX` を取得できます。
サイトセブンは初回だけ専用のログイン画面をブラウザで開き、ログイン後ページや対象店舗ページを確認できた時点で、その状態を `local_data` 配下へ保存して次回起動時以降も使い回します。
サイトセブンのデータ取得時に使うブラウザは、画面上で `表示` と `非表示` を切り替えられ、初期値は `表示` です。なお、ログイン操作そのものは毎回ブラウザを表示して開きます。
サイトセブン取得では、店舗ページへ直接飛ばずに `Top.do` から `福岡`、`春日市`、`Ａパーク春日店` を順に辿ってから対象機種ページへ進みます。
サイトセブン内のページ移動の間には、毎回 2 秒から 4 秒のゆらぎ待ちを入れています。
サイトセブン取得では、保存前に既存の `Aパーク春日店` を優先して寄せ先を決め、別店舗として増やさないようにしています。
サイトセブン取得では、機種ごとの設定ファイルをもとに `差枚` を自動計算し、四捨五入した整数として `累計ゲーム`、`BB回数`、`RB回数`、`合成確率`、`BB確率`、`RB確率` とあわせて既存と同じ保存形式へ並べます。`出率` は引き続き空欄です。
機種ごとの差枚計算条件は `config/machine_difference_rules.json` に置いてあり、今は `ネオアイムジャグラーEX` を登録済みです。今後はこの設定へ機種を足すだけで同じ仕組みを使い回せます。
サイトセブン取得で過去日の同一機種データがすでに `Supabase` にある場合は、その日付を保存前にスキップします。
ただし、日本時間の当日分だけはサイトセブンの内容を優先し、既存の同一機種データを入れ直してから上書き保存します。
サイトセブン側で取れる日数は直近8日までです。
新しく環境を作る場合は、`apps/gui/requirements.txt` の導入後に `python -m playwright install chromium` も一度実行してください。
`Supabase` 保存には、`.env.local` の `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` を使います。
`apps/web` には、`Supabase` に保存された内容を `店舗一覧 → 機種一覧 → 台データ` の順で見られるサイトを用意しています。
機種一覧ページは店舗全体の最新日一日分ではなく、各機種ごとの最新記録日を基準に一覧を出します。
このサイトは `PC` と `スマホ` の両方で、台データを横長の比較表として見られます。
