# Webサイトの現状

## 画面構成

- `apps/web` に `Vercel` 向けのサイトを用意しています
- ページの流れは `店舗一覧 → 機種一覧 → 台データ` です
- `店舗一覧` では、保存済みの店舗名だけを表で見られます
- `店舗一覧` では、店舗URLだけを登録待ちとして追加できます
- Web上で店舗名は取得せず、登録待ちURLはGUIアプリの更新または定期取得で店舗名を補完します
- `機種一覧` では、各機種ごとの最新記録日を基準に、保存済みの機種をリンク付きの表で見られます
- `台データ` では、同じ台番のデータを日付ごとに比較できます

## データの読み出し

- データ元は `Supabase` の `stores` と `machine_daily_results` です
- `apps/web` は `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` を使って読み出します
- 店舗URLの登録待ち追加では、`stores` にURLを保存します
- ローカル確認時は、ルートの `.env.local` も読めるようにしています
- 本番の `Vercel` では、同じ環境変数を `Vercel` 側に設定する想定です

## 表示について

- `PC` と `スマホ` の両方で、左端の日付列を固定した横長の比較表で見られます
- `スマホ` では店舗一覧、機種一覧、台データ比較の表を横に動かしながら確認します
- 店舗一覧は `stores` だけを読み、台データ行は読みません
- 機種一覧は、その店舗にある全機種の保存済み台データを読み、機種ごとに最も新しい日付の行だけを集計して表示します
- 機種一覧の台数、平均差枚、平均G数、平均出率は、それぞれの機種にとって最新の日一日分をもとに表示します
- 機種一覧は、まず機種ごとの最新日が新しい順、その次に台数の多い順で表示します
- サイトセブン由来で `Supabase` に差枚が入っていない行でも、機種ごとの設定ファイルと `G数`、`BB`、`RB` から `差枚` を補って表示します
- 差枚の計算条件はルートの `config/machine_difference_rules.json` にまとめてあり、今は `ネオアイムジャグラーEX` を登録済みです
- 台データ比較では、横幅を抑えるために表内の日付を `YY/MM/DD` 形式で表示し、日付の隣に曜日を表示し、数値の桁区切りを省いています
- 台データ比較では、チェックボックスで表示する列を選べます
- 台データ比較の `出率`、`BB率`、`RB率` は初期状態では非表示です
- アイムジャグラーEX系、スマスロ ハナビ、新ハナビ、スマスロ サンダーV、バーサスリヴァイズ、ハナハナホウオウ、ファンキージャグラー2、マイジャグラーV、ハッピージャグラーV、ゴーゴージャグラー、ジャグラーガールズ、ウルトラミラクルジャグラー、ミスタージャグラーの台データ比較では、`G数`、`BB`、`RB` から別アプリと同じ計算方法で設定を推測し、`設定` 列に加重平均値を表示します
- 対応機種の台データ比較では、`データ推測`、`G数による推測`、`特定日6あり` の比重を入力できます
- `G数による推測` は標準で `6000G` から `9000G` までを指数 `1.5` で評価し、指数を大きくすると後半ほど評価が伸びやすくなります
- `特定日6あり` は、店舗ごとに保存している特定日行だけ、その機種の中で最も良い台を高く見る相対評価として使います
- `特定日6あり` の比較に使う元の評価は、`G数による推測` がオフならデータ推測だけ、オンならデータ推測とG数推測を合わせた値です
- 機種名に `ジャグラー` を含む対応機種では、初期値はデータ推測100%、G数による推測と特定日6ありをオフにします
- ジャグラー以外の対応機種が8台以下の場合、初期値はデータ推測20%、G数による推測40%、特定日6あり40%で、G数と特定日6ありをオンにします
- ジャグラー以外の対応機種が9台以上の場合、初期値はデータ推測80%、G数による推測20%、特定日6あり0%で、特定日6ありをオフにします
- 対応機種では、該当台番の表示中の列にポインターを合わせると各設定の推測割合を確認できます
- 対応機種では、推測値が4.0以上は黄色系、4.5以上は緑系、5.0以上は赤系の単色背景と太字で、該当日の該当台番にある表示中の列を強調します
- 日付末尾やゾロ目による行の強調と設定推測による強調が重なった場合は、設定推測による強調を優先します
- 対応機種の台データページでは、設定別の `BIG確率`、`REG確率`、`合成確率` を表示します
- 台データ比較は表示列を減らすと列幅と文字を大きくし、横幅は5台分の表示を上限にします
- 台データ比較の日付列は、列幅を広げても日付が見切れないよう固定幅を確保します
- 台データ比較では、台番と台番の境界が分かるように、各台番の右端の縦線を少し太く表示します
- 台データページでは、日付末尾の `0` から `9`、`ゾロ目`、曜日を複数選び、該当日だけに絞るか、全日を表示したまま該当行だけを強調できます
- 曜日は日曜から土曜まで7つあり、初期状態ではすべて未選択です
- 台データページで選んだ特定日の末尾、ゾロ目、曜日は、店舗ごとの `stores` の設定として自動保存し、次回以降の表示に使います
- 曜日を保存するには、`stores` に `event_weekdays` の数値配列列を用意します。列がない環境でも画面上の切り替えはできますが、曜日だけは再読み込み後に初期状態へ戻ります
- 台データページの表示方法は初期状態で `強調` になり、必要に応じて `絞込` に切り替えられます
- 台データページの絞り込みと強調は、ページ全体を読み込み直さずに画面内で切り替えます
- 台データ比較の表は、「CSV出力」ボタンで表示中の列をCSVファイルとしてダウンロードできます
- 台データ比較のCSVには `曜日` 列と `特定日` 列を出し、画面で選んだ末尾、ゾロ目、曜日に該当する日は `はい`、それ以外の日は `いいえ` として確認できます
- 日付は保存済みの `YYYY-MM-DD` を基準に扱い、環境による時差で前日にずれないようにしています
- 各ページには前の一覧へ戻るための導線を置いています

## 軽量化について

- `Supabase` から同じ条件で読み出した結果は、標準で60秒だけ使い回します
- 取得結果の使い回し時間は、必要に応じて `HALLDATA_FETCH_CACHE_TTL_MS` でミリ秒単位に変更できます
- 外部書体の読み込みをやめ、端末に入っている日本語書体を使います
- 重いぼかし背景と初期表示時の動きを外し、表の描画を優先しています

## 実行方法

1. `apps/web` で `npm install` を実行する
2. `apps/web` で `npm run dev` を実行する
3. 必要に応じて `npm run build` で本番用の確認をする
