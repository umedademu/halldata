# GUIアプリの現状

## 対象店舗

- データ取得タブでは、対象店舗や店舗URLは入力しません
- 対象店舗は `登録店舗` タブの登録済み一覧で、取得対象のチェックを入れて選びます
- 初期状態では、登録済みの全店舗に取得対象のチェックが入っています
- 複数店舗にチェックを入れて取得すると、登録済み一覧の並び順で一店舗ずつ指定期間の台データを取得します
- `登録店舗` タブでは、店舗URLを入力するとページから店舗名を自動取得して登録できます
- `登録店舗` タブには `サイトセブン` `都道府県` `地域` `SS店舗名` の入力欄と一覧列があります
- `都道府県` の初期値は `福岡県`、`SS店舗名` の初期値は取得した店舗名です
- 登録済み一覧で1行選ぶと、その行の内容を入力欄へ読み戻して更新できます
- 登録した店舗一覧は `Supabase` に保存され、次回起動時も `Supabase` から読み込みます
- 店舗URLは保存前に表記ゆれをそろえるため、同じURLの大文字小文字違いなどで重複登録されにくくしています
- 登録店舗が1件も無い場合は、登録済み一覧は空のまま表示します

## 取得の流れ

1. `登録店舗` タブで取得対象にする店舗にチェックを入れる
2. 直近日数を入力する。初期値は `90` 日で、日本時間の今日までを対象にする
3. 必要なら `再試行休止`、`取得後に台データ表を表示しない`、`取得完了時に音を鳴らす` を切り替える。`再試行休止` は初期値 `10` 秒、`取得後に台データ表を表示しない` は初期状態でオン
4. 取得ボタンを押す
5. 取得前に、同じ店舗、同じ日付が全機種取得済みとしてローカル索引にあるかを確認し、見つかった日付は丸ごとスキップする
6. 取得中は、状態欄の下にある進捗バーで日付ごとの確認と機種ごとの取得の進み具合を見られる
7. 対象期間の確認や日付ごとの取得に失敗した場合は、指定した秒数だけ休止してから最大3回まで再試行する
8. 取得中は、取得ボタンの右隣にある `中止` ボタンで処理を止められる
9. 中止した場合は、現在の処理が区切れた時点で止まり、次の店舗へは進まない
10. 各日付の取得が終わるたびに、その日分だけローカル保存と `Supabase` 保存を自動で行う
11. 複数店舗を取得した場合は、チェックした店舗を順番に処理する
12. 台データ表を表示する設定の場合、複数店舗取得では最後に取得した店舗の表を画面へ表示する

## サイトセブン取得

- `データ取得` タブには `サイトセブン` の専用欄があり、登録店舗タブで `サイトセブン` 列にチェックを入れた店舗のジャグラー系対象機種をまとめて取得できます
- 初回起動時に、保存済みのログイン情報が無ければ、サイトセブンへログインするか確認します
- `サイトセブンにログイン` を押すと、指定されたログイン画面をブラウザで開きます
- ログイン後ページや対象店舗ページが見えたことを確認できると、その場でログイン状態を `local_data/site7_browser` に保存します
- 次回起動時以降は、その保存済みログイン状態を使うため、毎回の再ログインは不要な想定です
- ログイン後は、ブラウザを閉じなくても数秒で自動反映される想定です
- `取得時のブラウザ` は `表示` と `非表示` を切り替えられ、初期値は `表示` です
- この切替はサイトセブンのデータ取得時だけに使い、ログイン操作は毎回ブラウザを表示して開きます
- 選んだ表示設定はローカル設定として保存し、次回起動時も引き継ぎます
- `表示` で取得した場合は、取得完了後もそのブラウザを自動では閉じません
- `表示` のまま続けてサイトセブン取得を押した場合は、前回の表示ブラウザをそのまま使って次の取得を始めます
- `非表示` で取り直す時、ログイン操作を始める時、アプリを終了する時は、前回の表示ブラウザを閉じます
- 店舗ページへは直接飛ばず、`Top.do` から登録店舗ごとの `都道府県` `地域` `SS店舗名` を使って地区ページを経由して開きます
- 既知の店舗名に当てはまる場合は、`SS店舗名` が半角寄りでも内部で住所などの補助情報を補って照合します
- `SS店舗名` や地区リンクの照合では、全角と半角、空白、ハイフン、記号、末尾の `店` などの揺れをある程度吸収して比較します
- サイトセブン内のページ移動の間には、毎回 2 秒から 4 秒のゆらぎ待ちを入れます
- `サイトセブン取得` の右隣には専用の `中止` ボタンがあり、ページ遷移の合間やゆらぎ待機中ならそのまま停止できます
- `サイトセブン取得` は、データ取得タブの `直近日数` と `再試行休止` を使います
- サイトセブンは直近8日までしか取れないため、`直近日数` が8を超える場合は確認のうえで8日に切り詰めます
- 保存の優先順位は `みんレポ` が上です。サイトセブンは、同じ日付と機種のみんレポ由来データがまだ無い場合だけ仮置きします
- すでにみんレポ由来のデータがある日付と機種には、サイトセブン取得では上書きしません
- 同じ日付と機種にサイトセブン由来の仮置きが残っている場合だけ、サイトセブン取得で入れ直します
- あとからみんレポを取得した場合は、その内容でサイトセブンの仮置きより優先して上書きされます
- 新しい実行環境では、`requirements.txt` を入れたあとに `python -m playwright install chromium` を一度実行します
- サイトセブン取得では、機種ごとの設定ファイルをもとに `差枚` を自動計算し、四捨五入した整数として `累計ゲーム`、`BB回数`、`RB回数`、`合成確率`、`BB確率`、`RB確率` を既存と同じ保存形式へ並べ替えます
- `出率` は引き続き空欄として扱います
- 機種ごとの差枚計算条件は `config/machine_difference_rules.json` にまとめており、今は `ネオアイムジャグラーEX`、`SアイムジャグラーＥＸ`、`マイジャグラー`、`ゴーゴージャグラー`、`ファンキージャグラー`、`ミスタージャグラー`、`ジャグラーガールズ`、`ウルトラミラクルジャグラー`、`ハッピージャグラー` を登録済みです
- サイトセブンの対象機種判定は完全一致だけでなく、設定ファイルに置いた対象語を含むかどうかでも見ます
- 保存先は既存と同じで、`local_data` の `json` と `Supabase` の `stores`、`machine_daily_results` を使います
- 保存前に、既存データ件数が最も多い `Aパーク春日店` または `GOGOアリーナ天神` の店舗情報へ寄せ直し、サイトセブン側の店舗表記で別店舗を増やさないようにします
- 保存時は、`ゴーゴージャグラー３` や `ファンキージャグラー２ＫＴ` のような既存表記へ機種名も寄せ直し、表記ゆれで別機種を増やさないようにします
- ただし `ネオアイムジャグラーEX` と `SアイムジャグラーＥＸ` は別機種として分けて保存します
- この2機種は、最新取得で `SアイムジャグラーＥＸ` と確定した台番があれば、その台番の過去データも同じ機種へ移動させ、`ネオアイムジャグラーEX` 側からは外します
- 過去日の同一機種データがすでに `Supabase` にある場合は、その日付を保存前にスキップします
- 台データ表へ表示するときは、機種設定があるものは計算済みの `差枚` を表示し、`出率` だけ空欄として扱います

## 定期実行

- データ取得タブの `毎日 [ ] 時に実行` に `0` から `23` の時刻を入れて `設定` を押すと、アプリ起動中は毎日その時刻に取得します
- 初期状態では毎日 `2` 時に実行します
- `設定` を押した時刻はローカル設定として保存し、次回起動時も同じ時刻を引き継ぎます
- 定期実行では、最初に `Supabase` から登録店舗一覧を読み込み、その最新一覧を全て取得対象にしてから取得します
- 定期実行でも、データ取得タブの `再試行休止` に入っている秒数を使って、取得失敗時に最大3回まで再試行します
- 定期実行中に別の処理が動いている場合は、その処理が終わるまで待機します
- `解除` を押すと定期実行を止めます
- アプリを完全に終了すると、設定した定期実行も止まります

## 全機種取得について

- 取得時は、日付ページから機種一覧を内部で読み込み、その日の全機種を対象にします
- 日本時間の今日が 4/14 の場合は 4/14 までを対象にし、4/14 のページがまだ無ければ 4/13、4/12、4/11 のように過去側へ戻って最新のページを使います
- 複数台設置の機種と、一台設置の機種をどちらも一覧に含めます
- 機種選択や機種一覧の事前読込は行わず、取得ボタンだけで処理を始めます

## 台データ比較について

- 取得結果は、1列目を日付列として表示します
- `台データ表を広く表示` ボタンを押すと、他の項目をたたんで台データ表だけを大きく表示できます
- `日付末尾` のプルダウンで `0` から `9` を選ぶと、その数字で終わる日だけに絞って表示できます
- 横にスクロールしても、左端の日付列は固定したまま表示します
- 日付列は右側の表と横の余白なしで続けて表示します
- 日付列の行を押すと、その日付の横一列を強調表示できます
- 取得中は、状態欄の下にある進捗バーで全体の進み具合を確認できます
- `取得完了時に音を鳴らす` は初期状態でオンになっており、取得完了時に音で知らせます
- `取得後に台データ表を表示しない` は初期状態でオンになっており、取得結果の保存は行いつつ、重い比較表の描画だけを省略できます
- 後でチェックを外すと、直前に取得した結果から台データ表を表示できます
- 再描画後も、表の表示位置と上下のスクロールバーがずれにくいように調整しています
- 新しい全機種取得で保存済みの同一日付は取得時に日付ごとスキップし、概要欄にスキップ日数を出します
- 指定した直近日数の終了日データがまだ無い場合でも、見つかる最新日付まで戻って取得します
- 1行目の見出しには台番を並べます
- 2行目の見出しには、その台ごとの `機種名` `差枚` `G数` `出率` `BB` `RB` `合成` `BB率` `RB率` を並べます
- 行は日付ごとに並ぶため、同じ台の流れを日をまたいで見比べられます
- たとえば `7` を選ぶと、`3/7` `3/17` `3/27` `4/7` のような日だけを並べられます
- 台データ比較の各子見出しも、押すたびに昇順と降順を切り替えられます
- 表示が重くなりにくいよう、台データ比較は軽い描き方に見直しています
- 指定期間の中で機種が見つからない日は、その日の欄を空欄のまま残します

## 自動保存について

- 取得中は、各日付の取得が終わるたびに、その日分を `local_data` 配下へ `json` で自動保存します
- 全機種取得として保存した日付は、次回以降に大きい台データを開かずに済むよう、軽いローカル索引へ記録します
- あわせて、その日分を `.env.local` の設定を使って `Supabase` の `stores` と `machine_daily_results` に自動保存します
- サイトセブンの再取得時は、同じ `日付 + 台番` を単位に既存行を見ます
- 既存行が `みんレポ` のものなら残し、既存行が `サイトセブン` のものだけを入れ直します
- 次回以降の取得では、このローカル索引を見て、全機種取得済みの日付を丸ごと飛ばします
- `.env.local` では最低限 `SUPABASE_URL` と `SUPABASE_SERVICE_ROLE_KEY` を使います
- `.env.local` が未設定、または `Supabase` 保存に失敗した場合でも、取得済みの表表示は残します
- `.env.local` と `local_data` は追跡対象から外す想定です

## 登録店舗タブ

- 店舗URLを入力すると、ページから店舗名を自動取得して一覧へ登録できます
- 登録済み一覧の `取得対象` 欄を押すと、取得する店舗をチェック形式で切り替えられます
- 登録済み一覧の `サイトセブン` 欄を押すと、サイトセブン取得に使う店舗をチェック形式で切り替えられます
- `都道府県` `地域` `SS店舗名` は入力欄で自由に入れられ、`選択行を更新` で一覧へ反映できます
- `全て取得対象にする` と `取得対象を全て外す` で、一括してチェックを切り替えられます
- `最新に更新` を押すと、`Supabase` から最新の登録店舗一覧を読み込みます
- 登録済み一覧で行を選び、`選択した店舗を削除` を押すと、その店舗を `Supabase` から削除できます
- 店舗削除では、その店舗にひもづく `machine_daily_results` の台データも一緒に `Supabase` から削除します
- 店舗削除では、`local_data` 配下のローカル保存ファイルは削除しません
- WebアプリからURLだけで追加された登録待ち店舗は、`最新に更新` または定期実行時にGUIアプリが店舗名を取得して登録状態を補完します
- 新しく登録した店舗は、登録直後から取得対象に入ります
- 同じ店舗名または同じURLは重複登録しないようにしています
- 登録後は `Supabase` の `stores` に保存します
- 次回起動時は、`Supabase` の `stores` から登録店舗一覧を読み込んで表示します

## 常駐

- 画面右上の閉じるボタンを押してもアプリは終了せず、ウィンドウを非表示にして Windows の通知領域に常駐します
- 通知領域アイコンのメニューから、画面の再表示とアプリ終了ができます
- 常駐中も、設定した定期実行は動き続けます
